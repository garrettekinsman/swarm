# PBAR — Population-Based Annealed Research

Autonomous research loop that runs parallel AI research branches, applies softmax selection with temperature annealing, and iterates toward the best coverage.

## What It Is

PBAR is a structured multi-branch research protocol enforced by LangGraph. It's inspired by Population-Based Training (PBT) from ML optimization, adapted for open-ended research tasks.

**When to use:**
- Deep research questions requiring multi-angle coverage (geopolitical, technical, financial)
- Any topic where you want to find non-obvious signals, not just the most dramatic headlines
- Research that benefits from iterative refinement over 5+ generations

**Not for:** quick tactical lookups or single-question searches. For those, just use web_search directly.

## How It Works

### 4 Parallel Branches

Each generation runs 4 research branches simultaneously. Each branch has a query targeting a different angle of the topic (e.g., supply chain, prices, policy, infrastructure).

### Softmax Selection with Temperature Annealing

After each generation, branches are scored 0.0–1.0 on specificity and coverage. Softmax selection picks the top 2 — with temperature controlling exploration vs exploitation:

| Generation | Temperature | Behavior |
|------------|------------|----------|
| 1 | 2.00 | High exploration — random-ish selection |
| 2 | 1.20 | Moderate exploration |
| 3 | 0.72 | Starting to exploit |
| 4 | 0.43 | Strong exploitation |
| 5 | 0.26 | Near-greedy — best branches dominate |

Decay factor: `temp *= 0.6` per generation.

### Evolution

Top 2 branches survive and get refined queries. 2 mutant branches are generated by cross-pollinating query elements from the top performers. This keeps diversity while exploiting what's working.

### LangGraph Enforcement

The graph topology IS the contract. Nodes cannot be skipped:

```
initialize → search_all_branches → score_branches → softmax_select → evolve_branches → check_done
                                                                                            |
                                                         (gen < max) <--------------------+ 
                                                              |
                                                    search_all_branches
                                                    ...
                                                    (gen == max)
                                                         |
                                                     synthesize → END
```

## Budget

- ~$0.15–0.18/sprint (Sonnet orchestrator + Brave API searches)
- 5 generations ≈ $0.75 total
- 20 search paths per run (5 gens × 4 branches)

## Key Lesson

**Annealing optimizes for excitement, not coverage.** Pure PBAR will converge on the most dramatic/high-scoring angle by Gen 2-3, potentially missing "boring but critical" data. Use locked anchor branches (v2) to force coverage of must-have topics.
