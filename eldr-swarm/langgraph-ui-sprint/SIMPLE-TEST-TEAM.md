# Simple Test Team — Web Swarm
*v1 — 2026-03-24*

## Purpose
Validate the LangGraph agent-as-node pattern by building a simple EldrChat landing page.
Cheaper/faster than the full iOS pipeline. If this works, the pattern is proven.

## Team

| Agent | Model | Role |
|-------|-------|------|
| Orchestrator | claude-opus-4-6 (cloud) | Drives the graph, writes brief, routes work |
| GARRO | kimi-k2.5 via OpenRouter | Design spec: layout, colors, copy direction |
| Coder | qwen3-coder-next (Framework1 local) | Implements HTML/CSS from GARRO's spec |
| Vera | claude-sonnet-4-6 (cloud) | Security audit: no sketchy JS, no hardcoded secrets |

## Execution Rules (NON-NEGOTIABLE — learned 2026-03-24)

1. **Gaho never runs the loop.** Gaho spawns an Opus orchestrator sub-agent and yields.
2. **No exec loops in main session.** Main session = thin dispatch layer only.
3. **Framework1 = one inference at a time.** Coder node runs sequentially — no parallel local calls.
4. **Orchestrator must know the constraint.** Inject `FRAMEWORK1_CONSTRAINT` into orchestrator system prompt.

## How to Run (NON-NEGOTIABLE)

```
# WRONG — NEVER do this (locks main chat for 10+ min):
exec(command="python3 web_swarm.py")         ← BANNED

# RIGHT — Gaho spawns Opus orchestrator sub-agent, then yields immediately:
sessions_spawn(
    task="...",
    model="anthropic/claude-opus-4-6",
    mode="run",
    streamTo="parent"
)
sessions_yield()   ← Gaho is DONE, orchestrator takes over
```

Gaho's job is dispatch only. The orchestrator sub-agent runs the graph,
monitors progress, and reports back. Gaho never polls or waits in a loop.

## Lessons Learned

- **Sanitizer signature:** `sanitize_text(value, tier=...)` not `sanitize_text(field, value, tier=...)`
- **Regex for code extraction:** LLM wraps in ```html ... ``` — strip fences before writing to file
- **kimi-k2.5 via OpenRouter:** works well for design specs, verbose but high quality
- **qwen local output:** good code, but slow — budget 3-5 min for coder node
- **Main session hygiene:** this test locked main chat for ~10 min — never again
- **Model router response key:** router returns `"result"`, NOT `"content"` — `data.get("result", data.get("content", ""))` is the safe pattern
